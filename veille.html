Rust Blog 2024
Pourquoi Rust remplace C++ dans les projets critiques ?

Rust propose un modèle de mémoire unique basé sur l'ownership et le borrow checker qui empêche les erreurs mémoire à la compilation. Des entreprises comme Microsoft, Google et Mozilla l'ont adopté pour remplacer du code C/C++ dans leurs infrastructures critiques.

Ce que j'en retiens La sécurité mémoire sans garbage collector est révolutionnaire. Rust force à écrire du code correct dès la compilation, ce qui réduit drastiquement les vulnérabilités de type buffer overflow.

Rust : langage le plus apprécié depuis 9 ans consécutifs

Pour la 9e année consécutive, Rust arrive en tête du classement des langages les plus aimés par les développeurs dans le sondage annuel de Stack Overflow. Sa communauté croît fortement et l'écosystème Cargo / crates.io est très mature.

The Verge / Linux 2023 — 2024
Rust entre dans le noyau Linux — un tournant historique

En 2022, Linus Torvalds a accepté d'intégrer Rust comme second langage officiel du noyau Linux. Les premiers modules Rust sont arrivés dans Linux 6.1. Cela représente une rupture majeure après 30 ans de domination exclusive du C.

Ce que j'en retiens Si Rust est assez bon pour le noyau Linux, c'est que ses garanties de sécurité sont réelles et prouvées. Cela valide l'approche et montre l'avenir du développement système.

MDN / Dev.to 2024
REST vs GraphQL vs gRPC : quelle architecture API choisir ?

Les trois grandes architectures d'APIs ont chacune leurs forces. REST est simple et universel, GraphQL permet au client de choisir ses données, et gRPC excelle en performance pour les communications entre microservices.

Ce que j'en retiens Pour les projets web classiques, REST reste le standard. GraphQL devient intéressant quand on a des besoins de données complexes côté front. gRPC est à connaître pour les architectures microservices.

OWASP 2023 — 2024
OWASP API Security Top 10 — Les failles à connaître

L'OWASP publie le classement des 10 vulnérabilités API les plus critiques. Parmi elles : l'authentification cassée, l'exposition d'objets (BOLA), l'injection, et la mauvaise gestion des accès. Des failles très courantes dans les APIs PHP que je développe.

Ce que j'en retiens Sécuriser une API n'est pas optionnel. La connaissance de ces failles m'aide directement dans mes projets PHP/PDO à écrire du code plus sûr dès le départ.

Postman / Swagger 2024
OpenAPI 3.1 et la documentation automatique des APIs

La spécification OpenAPI (anciennement Swagger) permet de décrire une API REST de façon standardisée. Postman, Insomnia et Swagger UI génèrent automatiquement une documentation interactive lisible par humains et machines.

Ce que j'en retiens Documenter ses APIs avec OpenAPI est une compétence professionnelle clé. Cela facilite la collaboration en équipe et montre la rigueur du développeur. Outil à maîtriser pour les projets BTS.
